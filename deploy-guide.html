01 · Local Run

Running on your machine

From zero to a running local server in under 5 minutes.

1
Clone the repository
Get the code onto your machine.
terminal
git clone https://github.com/yourusername/argus.git
cd argus
2
Install dependencies
Create a virtual environment and install all packages. This takes 2–3 minutes the first time.
terminal
python -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
3
Configure environment
Copy the example env file. Adding your Anthropic API key is optional but enables richer LLM responses.
terminal
cp .env.example .env
# Edit .env and add your ANTHROPIC_API_KEY (optional)
4
Train model + build RAG index
Generates 50K synthetic training records, trains the XGBoost model with calibration, and builds the FAISS vector index from policy documents. Takes ~2 minutes.
terminal
python -m scripts.train_model --generate

# Output:
# ✓ Generated 50,000 records
# AUC-ROC : 0.9420
# Model saved → models/argus_model.joblib
5
Start the server
Launch FastAPI with auto-reload. The app opens at http://localhost:8000 — the UI, API docs, and all endpoints are live.
terminal
uvicorn backend.main:app --reload

# Visit:
# http://localhost:8000        → UI
# http://localhost:8000/api/docs  → Swagger
02 · GitHub Setup

Push your code to GitHub

GitHub is the source of truth. Hugging Face will pull from it automatically on every push.

1
Create a new GitHub repository
Go to github.com/new → name it argus → Public → No README (we have one) → Create.
2
Push the code
terminal
git init
git add .
git commit -m "feat: initial Argus platform"
git branch -M main
git remote add origin https://github.com/yourusername/argus.git
git push -u origin main
3
Verify .gitignore is working
The .gitignore excludes models/*.joblib and data/raw/*.csv — these are too large for GitHub and get regenerated at deploy time via the Dockerfile.
terminal — verify excluded
git status --short
# Should NOT see: models/ or data/raw/
03 · Hugging Face Spaces

Deploy to Hugging Face Spaces

Free forever. Never sleeps. The DS community's deployment platform of choice.

Your live URL will be
https://huggingface.co/spaces/yourusername/argus
Replace yourusername with your actual HuggingFace username.
1
Create a new Space
Go to huggingface.co/new-space
→ Space name: argus
→ License: MIT
→ SDK: Docker ← this is the critical choice
→ Hardware: CPU Basic (free)
→ Visibility: Public
→ Click Create Space
2
Add HF as a git remote
Hugging Face Spaces is a git repository. You push to it the same way you push to GitHub.
terminal
git remote add hf https://huggingface.co/spaces/yourusername/argus
git push hf main
HF will ask for your username and a token (not your password). Get your token from HF Settings → Access Tokens → New token → Write access.
3
Watch the build
HF reads your Dockerfile and builds the image. First build takes 5–8 minutes (downloading Python, installing deps, training model). Subsequent pushes take ~2 minutes as layers are cached.
Build log
Go to your Space → click the Logs tab to watch the Docker build live. If anything fails, the error will be here.
4
Your app is live
Once the build completes, your Space shows ● Running. Visit the URL and share it. Every git push hf main triggers a redeploy automatically.
future deployments — just two commands
git add . && git commit -m "update"
git push hf main && git push origin main
04 · Secrets & Environment Variables

Configure your API keys securely

Never commit secrets to git. HF Spaces has a built-in secrets manager.

1
Open Space Settings → Secrets
In your HF Space → Settings tab → scroll to Repository secrets section.
2
Add each secret
Click New secret for each variable below. Secrets are encrypted at rest and injected as environment variables at runtime.
Secret nameValueRequired
ANTHROPIC_API_KEYYour key from console.anthropic.comOptional
APP_ENVproductionOptional
LOG_LEVELINFOOptional
Works without secrets
The app runs fully without ANTHROPIC_API_KEY. The RAG and Agent layers use an intelligent rule-based extractor as fallback. Add the key when you want Claude-generated answers.

The included GitHub Actions workflow runs tests and pushes to HF Spaces automatically on every merge to main.

Add GitHub repository secrets
Go to your GitHub repo → Settings → Secrets and variables → Actions → New repository secret.

07 · Troubleshooting

Common issues

Most problems have a one-line fix. Check the HF Spaces Logs tab first.

Build fails — "No space left on device"
HF free tier has limited disk space. In the Dockerfile, reduce training samples: change --samples 20000 to --samples 10000.
Build fails — "killed" during pip install
Memory limit hit. Add --no-cache-dir to the pip install command in the Dockerfile. Also consider using the cpu-upgrade hardware tier ($0.05/hr).
App shows "model not loaded" on first request
The Dockerfile pre-trains the model during build so this shouldn't happen. If it does, check the build logs for training errors. The model auto-trains on first request as a fallback.
RAG returns "No documents found"
The policy files in data/policies/ weren't copied into the Docker image. Ensure they are NOT listed in .gitignore and are committed to git.
Slow first API call after deploy
Normal — the model is loading from disk and SHAP explainer is initialising. Subsequent calls are fast (<200ms). Add a health check ping in the Dockerfile's CMD to pre-warm on startup.
ANTHROPIC_API_KEY not being picked up
Secrets added in HF Space settings are injected as environment variables — they don't need to be in the Dockerfile. Restart the Space after adding a secret for it to take effect.
useful debug commands (run locally with Docker)
# Build image locally
docker build -t argus .

# Run locally on port 7860
docker run -p 7860:7860 -e ANTHROPIC_API_KEY=your_key argus

# Check logs
docker logs $(docker ps -q --filter ancestor=argus)
